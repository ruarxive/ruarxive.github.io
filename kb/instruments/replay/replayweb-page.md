# ReplayWeb.page

**ReplayWeb.page** — это браузерный, полностью клиентский движок воспроизведения для локальных и удалённых WARC & WACZ файлов.

## Описание

ReplayWeb.page работает полностью в браузере, не требуя серверного ПО. Это делает его идеальным для локального просмотра архивов.

### Особенности

*   **Браузерный**: Работает полностью в браузере
*   **Клиентский**: Не требует серверного ПО
*   **WARC & WACZ**: Поддержка обоих форматов
*   **Простота**: Просто откройте файл в браузере

## Использование

### Онлайн версия

Откройте [ReplayWeb.page](https://replayweb.page/) в браузере и загрузите WARC или WACZ файл.

### Desktop приложение

Скачайте Desktop приложение с [GitHub releases](https://github.com/webrecorder/replayweb.page/releases).

### Локальная установка

```bash
git clone https://github.com/webrecorder/replayweb.page
cd replayweb.page
npm install
npm start
```

## Работа с файлами

### Загрузка WARC файла

1. Откройте ReplayWeb.page
2. Нажмите "Load WARC file"
3. Выберите WARC файл
4. Дождитесь загрузки и индексации

### Загрузка WACZ файла

1. Откройте ReplayWeb.page
2. Нажмите "Load WACZ file"
3. Выберите WACZ файл
4. Файл откроется автоматически (WACZ содержит индексы)

### Удалённые файлы

Можно загружать файлы по URL:

```
https://replayweb.page/?source=https://example.com/archive.wacz
```

## Особенности

### Просмотр страниц

*   Навигация по заархивированным страницам
*   Поиск по URL
*   Календарь для выбора даты
*   Список всех заархивированных страниц

### Интерактивность

*   Работают ссылки между заархивированными страницами
*   Сохранение состояния навигации
*   История просмотра

## Сравнение с другими инструментами

| Инструмент | Требует сервер | Локальный просмотр | Простота | Рекомендация |
| :--- | :--- | :--- | :--- | :--- |
| **ReplayWeb.page** | ❌ | ✅ | ✅ Высокая | **Для локального просмотра** |
| **PYWB** | ✅ | ❌ | ⚠️ Средняя | Для веб-сервисов |
| **OpenWayback** | ✅ | ❌ | ⚠️ Низкая | Для серверных установок |

### Когда использовать ReplayWeb.page

*   Локальный просмотр архивов
*   Не нужен сервер
*   Быстрый просмотр WACZ файлов
*   Демонстрация архивов

### Когда использовать другие инструменты

*   Веб-сервис для множественных пользователей (используйте PYWB)
*   Высокая производительность (используйте OpenWayback)
*   Распределённое хранение (используйте ipwb)

## Best practices

### Работа с большими файлами

*   WACZ файлы более эффективны для больших архивов
*   Используйте индексацию для быстрого доступа
*   Разделяйте большие архивы на меньшие WACZ файлы

### Совместное использование

*   WACZ файлы легко делиться
*   Можно загружать на веб-сервер для доступа по URL
*   Desktop приложение для офлайн использования

## Ограничения

*   Работает только в современных браузерах
*   Большие WARC файлы могут быть медленными
*   Ограниченная поддержка некоторых типов контента
