Международный опыт сохранения цифрового наследия

Примеры успешных проектов по сохранению цифрового наследия из других стран, которые могут быть полезны для российского контекста.

End of Term Presidential Harvest 2016​

О проекте​

Проект "End of Term Presidential Harvest 2016" — это 5-месячная инициатива Библиотеки Конгресса США, нескольких университетов и библиотек по архивации всех сайтов уходящей администрации Барака Обамы.

Особенности​

  • Масштаб: Архивация всех сайтов федерального правительства США
  • Срок: 5 месяцев активной работы
  • Участники: Библиотека Конгресса, университеты, библиотеки
  • Инструмент номинации: Использование Nomination Tool, когда любой мог предложить ссылки для приоритетной архивации

Результаты​

  • Более 11 тысяч ссылок было предложено 294 участниками
  • Создан полный архив администрации Обамы
  • Сохранены все официальные сайты и документы

Уроки для России​

  1. Координация: Важность координации между различными организациями
  2. Краудсорсинг: Использование инструментов номинации для привлечения сообщества
  3. Планирование: Заблаговременная подготовка к архивации
  4. Ресурсы: Выделение достаточных ресурсов на проект

DataRefuge​

О проекте​

DataRefuge — проект, созданный специально для сохранения архивных государственных данных, находящихся под угрозой исчезновения.

Контекст​

Проект был создан в ответ на избрание Дональда Трампа президентом США в 2016 году. В его команде было много климатических скептиков, выступающих против борьбы с глобальным потеплением. Судьба агентства EPA (Агентство по охране окружающей среды США) и их данных была под вопросом.

Деятельность​

17 января 2017 года группа энтузиастов собралась на хакатон по сбору данных по экологии, опубликовав 18 наборов данных на портале DataRefuge.

Результаты​

  • Сохранены критически важные данные по экологии
  • Создана платформа для сохранения государственных данных
  • Привлечено внимание к проблеме сохранения научных данных

Уроки для России​

  1. Срочность: Важность быстрого реагирования на угрозы
  2. Специализация: Фокус на конкретных типах данных (экология, климат)
  3. Сообщество: Мобилизация энтузиастов для решения проблемы
  4. Платформа: Создание специализированной платформы для сохранения данных

Сравнение с российской ситуацией​

Сходства​

  • Проблема уничтожения данных: В России ситуация с уничтожением официальных сайтов ничуть не лучше
  • Отсутствие государственной поддержки: В России также отсутствует системная поддержка архивации
  • Важность гражданского общества: Решение проблемы требует участия гражданского общества

Отличия​

  • Масштаб проектов: В США проекты получают больше ресурсов и поддержки
  • Координация: В США лучше налажена координация между организациями
  • Планирование: В США больше внимания уделяется заблаговременному планированию

Проект "Архивация государства"​

Российский контекст​

Много лет назад началась работа по архивированию госсайтов, находящихся в зоне риска, а также перечня сайтов, которые могли бы исчезнуть, но представляют ценность.

Результаты​

  • Собрано несколько терабайт данных в сжатом виде
  • Десятки-сотни терабайт в распакованном виде
  • Создана база архивов государственных сайтов

Вызовы​

  • Ресурсы: Сложно привлечь средства на такой проект
  • Волонтёры: Оказывается очень немного волонтёров
  • Время: За деятельностью над открытыми данными и коммерческими проектами на архивацию остаётся не так много времени

Уроки для российского контекста​

Что можно применить​

  1. Инструменты номинации: Создание системы, где любой может предложить сайты для архивации
  2. Координация: Улучшение координации между различными организациями
  3. Специализация: Фокус на конкретных типах сайтов или данных
  4. Планирование: Заблаговременная подготовка к архивации

Что нужно учитывать​

  1. Ресурсы: В России меньше ресурсов, нужно искать эффективные решения
  2. Волонтёры: Необходимо привлекать больше волонтёров
  3. Контекст: Учитывать специфику российской ситуации
  4. Долгосрочность: Обеспечить долгосрочную поддержку проектов

Выводы​

  1. Международный опыт важен: Можно учиться на успешных проектах других стран
  2. Адаптация необходима: Нужно адаптировать подходы к российскому контексту
  3. Сообщество критично: Без активного сообщества проекты не работают
  4. Планирование важно: Заблаговременное планирование повышает эффективность
